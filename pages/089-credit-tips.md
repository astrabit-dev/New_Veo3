# 75. 크레딧 절약 10가지 팁

## 실전 절약 전략 모음

다음 10가지 팁은 VEO 3를 수백 시간 사용한 경험에서 추출한 가장 실용적인 절약 전략들입니다. 각 팁을 습관으로 만들면 같은 예산으로 2-3배 많은 영상을 만들 수 있습니다.

---

### 팁 1: 3단계 테스트 전략 — Fast → Lite → 고품질

**이유**: Veo 3.1 Lite 등장 이후 "Fast Mode → 바로 고품질"의 2단계 전략은 비효율적입니다. 3단계 전략으로 크레딧 낭비를 줄이면서 품질도 검증할 수 있습니다.

**방법**:
```
1단계: Fast Mode (~15크레딧)
   → 방향성 확인 (구도, 캐릭터, 배경이 의도대로 나오는가?)

2단계: Veo 3.1 Lite (~40크레딧)
   → 오디오 포함 중간 품질로 검증 (음향, 디테일이 맞는가?)
   → SNS 콘텐츠라면 이 단계가 최종일 수도 있음

3단계: Veo 3.1 고품질 (~90크레딧)
   → 완전히 방향이 검증된 프롬프트만 사용
   → 최종 납품·공개용 영상에만 한정
```

**Veo 3.1 Lite를 중간 단계로 쓰는 핵심 이유**: Fast Mode는 오디오가 없어서 실제 최종 영상과 분위기가 다를 수 있습니다. Lite로 오디오까지 확인한 뒤 고품질을 사용하면 불필요한 재생성을 줄입니다.

---

### 팁 2: 짧고 명확한 테스트 프롬프트

**이유**: 긴 프롬프트의 테스트는 어디에서 문제가 생겼는지 찾기 어렵습니다.

**방법**:
- 테스트 시에는 프롬프트를 50-70자로 압축
- 핵심 요소 3가지만 포함: "무엇", "어디서", "어떤 분위기"
- 문제 파악 후 디테일 추가

```
// 테스트용 간략 버전
"Samurai warrior in a bamboo forest, rain, dramatic"

// OK 확인 후 최종 버전
"A tall samurai in black lacquered armor stands motionless in a dense bamboo forest during heavy rain. Water streams down his kabuto helmet..."
```

---

### 팁 3: 성공한 프롬프트 반드시 저장

**이유**: 좋은 결과를 낸 프롬프트를 잊어버리면 다음에 처음부터 다시 테스트해야 합니다.

**방법**:
- 텍스트 파일이나 노션에 프롬프트 저장
- 생성된 영상과 함께 프롬프트 기록
- "작동함", "약간 수정 필요", "실패" 태그 추가

---

### 팁 4: 한 프롬프트로 여러 클립 활용

**이유**: 같은 프롬프트로 생성해도 매번 다른 결과가 나옵니다. 동일 프롬프트를 여러 번 실행해서 가장 좋은 것을 선택하세요.

**방법**:
- High Quality로 같은 프롬프트 2-3번 실행 (300크레딧)
- 가장 좋은 결과 1개 사용
- 나머지 2개는 다른 장면 편집에 활용 가능 여부 확인

---

### 팁 5: 조명은 영상의 절반

**이유**: 조명을 잘 설정하면 환경, 분위기, 캐릭터 묘사를 일부 생략해도 됩니다. 조명 하나로 많은 것을 전달할 수 있어 프롬프트 길이와 수정 횟수를 줄입니다.

**방법**:
- 조명 묘사에 가장 많은 시간 투자
- 검증된 조명 공식 사용: 골든아워, 드라마틱 언더라이팅, 싱글 스폿라이트

---

### 팁 6: 인물 클로즈업 최소화

**이유**: 얼굴 클로즈업은 VEO 3에서 가장 재생성이 많이 필요한 부분입니다. 눈, 입, 표정의 미세한 부자연스러움이 눈에 띄기 때문입니다.

**방법**:
- 가능하면 미디엄샷 이상 사용
- 클로즈업이 필요하다면 "시선을 아래로" 또는 "실루엣" 처리
- 손, 발, 등 클로즈업으로 대체 가능한지 검토

---

### 팁 7: 복잡한 물리 효과 피하기

**이유**: 물, 불, 연기, 폭발 같은 물리 시뮬레이션은 VEO 3가 제어하기 어려워 재생성이 필요할 가능성이 높습니다.

**방법**:
- 주요 장면에만 물리 효과 사용
- "subtle smoke wisps" (약한 연기)가 "massive explosion" (거대한 폭발)보다 안정적
- 효과를 "subtle", "gentle", "slight"로 조절

---

### 팁 8: 환경은 구체적으로, 캐릭터는 명확하게

**이유**: VEO 3는 구체적인 지시를 더 잘 따릅니다. 모호한 설명은 예측 불가능한 결과를 낳고 재생성이 필요합니다.

**방법**:
- "a forest" → "a dense pine forest with thick undergrowth and fallen logs"
- "a woman" → "a Korean woman in her 30s with short black hair wearing a red coat"

---

### 팁 9: 배치 생성 계획

**이유**: 같은 세계관, 같은 캐릭터의 여러 클립을 만들 때 한 번에 계획해서 공통 요소를 미리 테스트하면 개별 수정 횟수가 줄어듭니다.

**방법**:
- 시리즈 영상 계획 시 1편의 캐릭터/환경/조명 설정을 먼저 확정
- 설정이 확정되면 이후 클립은 그 설정을 그대로 재사용

---

### 팁 10: 목적에 맞는 길이 선택

**이유**: 8초 클립이 항상 필요한 건 아닙니다. 짧은 장면에는 짧은 클립이 더 효율적일 수 있습니다.

**방법**:
- Google Flow에서 제공하는 경우 4-5초 클립 옵션 활용 (더 낮은 크레딧)
- CTA 엔딩이나 로고 클로즈업은 4초면 충분
- 긴 움직임이 필요한 트래킹샷에만 8초 사용

---

### 팁 11: SNS 콘텐츠는 Veo 3.1 Lite로 완결

**이유**: TikTok, Instagram Reels, YouTube Shorts처럼 짧고 빠르게 소비되는 플랫폼에서는 Veo 3.1 고품질과 Lite의 체감 차이가 작습니다. 모바일 화면과 빠른 스크롤 환경에서 시청자는 미묘한 화질 차이보다 **콘텐츠 자체**에 반응합니다.

**방법**:
- TikTok / Reels / Shorts 전용 콘텐츠: Veo 3.1 Lite로 최종 완성
- YouTube 장편 / 클라이언트 납품 / 포트폴리오: Veo 3.1 고품질 사용
- 결과: 동일 크레딧으로 SNS 콘텐츠 제작량 **2배 이상** 증가

---

## 크레딧 절약 효과 요약

| 팁 | 예상 절약 효과 |
|----|-------------|
| **3단계 테스트 전략** | **40-60%** ← 가장 중요 |
| 짧은 테스트 프롬프트 | 20-30% |
| 성공 프롬프트 저장 | 장기적으로 30%+ |
| 조명 최우선 투자 | 15-25% |
| 클로즈업 최소화 | 10-20% |
| 구체적인 묘사 | 20-30% |
| 배치 계획 생성 | 25-35% |
| **SNS는 Lite로 완결** | **월 SNS 클립 2배** |

11가지 팁을 모두 적용하면 같은 결과를 내는 데 드는 크레딧을 **최대 60-75% 절약**할 수 있습니다.
